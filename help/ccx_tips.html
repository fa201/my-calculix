General

Installation and set up

For Linux Ubuntu, copy ccx program from the website and paste at /usr/local/bin. Then make an alias to this program to launch directly from the folder of the study. Make sure the program has execute right (7xx with chmod)

To switch from 1 to 4 CPU, export OMP_NUM_THREADS=4.

Units

This is a source of mistake so I always work with the same system: mm, ton sec.

  • length: mm
  • mass: ton
  • time: s
  • energy, work: mJ
  • mass per volum: ton/mm3
  • stress, pressure: MPa
  • force: N
  • acceleration: mm/s2
  • velocity: mm/s
  • frequency: Hz
This gives for steel: E = 210 000 MPa, rho= 7.8e-9.

Information retrieved from YahooGroup

Shape functions are those from text books. No ammendments are performed for locking phenomena.

Contacts: Face-to-face contact should preferably be used with the linear pressure-overclosure behavior. The only constant in that case is the spring constant. Use of *FRICTION further requires a friction coefficient and a stick slope. For steel I usually take 1.e7 MPa/mm for the spring stiffness and 1.e5 MPa/mm for the stick slope.

Merge *.inp files with ID offset: read ccx-file inp add

DLOAD on 3D element: the pressure is always directed into the body. This does not depend on whether the face is reflective or black.

*FREQUENCY works only for 3D elements.

For quadratic elements, stress at midpoints are obtained by averaging the end point strssses.

Beam expansion: if you do not use OUTPUT=3D you get the mean stress over the thickness, which is a very bad approximation for beams under bending.

All elements in CalculiX use the textbook shape functions, i.e. no amendments are performed for locking phenomena.

Error estimator: values are available internally.

Output format: binary files (*NODE OUTPUT) have a better accuray than ASCII files (*NODE FILE). Source.
